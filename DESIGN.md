# DESIGN.md

Treat this file and `UI_UX_GUIDELINES.md` as the source of truth for all product, game design, and visual design decisions.

## Product name

**Big2Go**

## Brand direction

Big2Go should feel like a real mobile card game: fast, polished, simple to understand, and easy to start with one thumb.

## Landing page hierarchy

1. Top status/settings bar
2. Large card/logo hero
3. Dominant gold **PLAY NOW** button
4. Three quick actions: Continue, Rules, Share
5. Player selector: 4 / 3 / 2 players
6. Bottom utility buttons: Rank, Bonus, Goals

## Visual style

- Deep purple casino/card-table background
- Gold CTA button
- Bright card symbols
- Rounded mobile-game panels
- Large touch targets
- Minimal words on the first screen

## Mobile UX rules

- The first screen should fit in one mobile viewport whenever possible.
- Avoid long explanations on the landing page.
- Put rules and extra details behind buttons/dialogs.
- Keep all visible landing controls clickable.
- Avoid horizontal page overflow.

## Naming rules

Use:

- Visible name: `Big2Go`
- Slug/file prefix: `big2go`

Do not use old names:

- `Lucky2`
- `lucky2`
- `Big Two Moon Market`
- `big-two-moon-market`
